#### <a name="how-do-i-upgrade"></a><span data-ttu-id="e160d-195">¿Cómo puedo actualizar?</span><span class="sxs-lookup"><span data-stu-id="e160d-195">How do I upgrade?</span></span>

<span data-ttu-id="e160d-196">Mientras se encontrará detalles acerca de cómo actualizar en el conjunto de contenido de la [actualización a Project Server 2010](https://go.microsoft.com/fwlink/p/?linkid=841812) , es importante saber que existen dos métodos distintos que puede usar para actualizar:</span><span class="sxs-lookup"><span data-stu-id="e160d-196">While details on how to upgrade can be found in the [Upgrade to Project Server 2010](https://go.microsoft.com/fwlink/p/?linkid=841812) content set, it is important to understand that there are two distinct methods you can use to upgrade:</span></span> 
  
- <span data-ttu-id="e160d-p114">**Actualización de base de datos adjunta:** Este método sólo actualiza el contenido para su entorno y no los valores de configuración.   - <span data-ttu-id="e160d-200">**Actualización completa de base de datos adjunta** : migra los datos del proyecto almacenados en las bases de datos de Office Project Server 2007, además de los datos del sitio de Microsoft Project Web App (PWA) almacenados en una base de datos de contenido de SharePoint.</span><span class="sxs-lookup"><span data-stu-id="e160d-200">**Database-attach Full upgrade** - Migrates the project data stored in the Office Project Server 2007 databases, plus the Microsoft Project Web App (PWA) site data stored in a SharePoint content database.</span></span>

- <span data-ttu-id="e160d-p115">**Actualización inmediata**: los datos de configuración para la granja de servidores y todo el contenido de la granja de servidores se actualizan en el hardware existente, en un orden fijo. Cuando se inicia el proceso de actualización en contexto, el programa de instalación toma la granja de servidores completa sin conexión y los sitios Web y sitios de Microsoft Project Web App no estarán disponibles hasta que finalice la actualización y, a continuación, el programa de instalación reinicia el servidor. Después de comenzar una actualización en contexto, no se puede poner en pausa la actualización o revertir a la versión anterior. Se recomienda realizar una imagen reflejada de su entorno de producción y para realizar la actualización en contexto a este entorno y no el entorno de producción.</span><span class="sxs-lookup"><span data-stu-id="e160d-p115">**In-Place upgrade**: The configuration data for the farm and all content on the farm is upgraded on the existing hardware, in a fixed order. When you start the in-place upgrade process, setup takes the entire farm offline and the Web sites and Microsoft Project Web App sites are unavailable until the upgrade is finished, and then setup restarts the server. After you begin an in-place upgrade, you cannot pause the upgrade or roll back to the previous version. It is highly suggested to make a mirrored image of your production environment and to do the in-place upgrade to this environment, and not your production environment.</span></span>
