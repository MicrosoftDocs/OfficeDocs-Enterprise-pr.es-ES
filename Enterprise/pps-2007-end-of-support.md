## <a name="what-does-end-of-support-mean"></a>¿Qué final hace de Media de soporte técnico?

Productos de Microsoft, por ejemplo, SP3 de PerformancePoint Server 2007, software de ProClarity y SharePoint Server 2007 SP3 tenían un ciclo de vida de soporte técnico durante el cual Microsoft proporciona nuevas características, correcciones de errores y las actualizaciones de seguridad. El ciclo de vida de un producto dura normalmente de 10 años desde la fecha de lanzamiento del producto y el final de dicho ciclo de vida se conoce como fin del producto del soporte. Como ProClarity, PerformancePoint Server y SharePoint Server 2007 han alcanzado su fin de soporte, Microsoft ya no proporcionará:
  
- Soporte técnico para problemas que puedan surgir
    
- Correcciones de errores para los problemas que se descubren y que puede afectar a la estabilidad y la facilidad de uso de servidores
    
- Revisiones de seguridad para las vulnerabilidades de seguridad que se descubren y que puede hacer que los servidores o aplicaciones vulnerables a las infracciones de seguridad
    
- Actualizaciones de zona horaria
    
La instalación de PerformancePoint Server 2007 SP3, SharePoint Server 2007 SP3 y ProClarity seguirá ejecutándose aunque haya finalizado el soporte técnico. Sin embargo, se recomienda encarecidamente que migrar desde estas aplicaciones tan pronto como sea posible.

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Usar Power BI (en la nube o local)

Power BI es un conjunto de herramientas de análisis de negocio para analizar los datos y compartir conocimientos. Con Power BI, puede crear informes interactivos y paneles mediante local u orígenes de datos en línea. Las personas pueden ver y utilizar los informes y paneles usando sus equipos o dispositivos móviles.
  
Power BI no se incluye en Office 365 o SharePoint Server, pero está ofreciendo un independiente que incluye Power BI escritorio, las puertas de enlace de Power BI y el servicio de Power BI. Power BI también se integra con SharePoint Online. Puede empezar a trabajar con Power BI de manera gratuita y en función de sus datos de uso y empresarial necesidades, actualización para profesionales de TI de Power BI. Para obtener más información, vea [¿Qué es Power BI?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Usar Reporting Services (local)

SQL Server Reporting Services proporciona una solución de informes sólida, junto con la capacidad de instalar y configurar Reporting Services en modo nativo o en modo integrado de SharePoint. Puede crear informes de uso de varias herramientas, como el Diseñador de informes, el generador de informes y Power View. Con la versión más reciente de SQL Server, también puede usar el Editor de informes de SQL Server Mobile para entregar informes que escalar a cualquier tamaño de pantalla, proporcionar a su organización la capacidad de usar informes en sus dispositivos móviles. Para obtener más información, vea [SQL Server Reporting Services (SSRS): crear, implementar y administrar informes de móviles y paginados](https://go.microsoft.com/fwlink/?linkid=841342).
  
### <a name="use-performancepoint-services-on-premises"></a>Uso de PerformancePoint Services (local)

Como ya sabe, PerformancePoint Server 2007 ha sido adquirido por separado en SharePoint Server 2007. A partir de SharePoint Server 2010, PerformancePoint Services es una aplicación de servicio en SharePoint Server. Esto significa que no es necesario comprar licencias de servidor independiente o de hardware para uso de PerformancePoint Services.
  
Para mover de PerformancePoint Server 2007 a PerformancePoint Services, mover a una versión más reciente de SharePoint Server y, configuración de PerformancePoint Services. La versión del servidor de SharePoint que desea mover a determinará si puede importar el contenido existente del panel de PerformancePoint Server 2007 a PerformancePoint Services.
  
- Si está actualizando a SharePoint Server 2010, puede importar el contenido del panel de PerformancePoint de PerformancePoint Server 2007 a PerformancePoint Services en SharePoint Server 2010. Para obtener más información acerca de cómo funciona esto, vea [Asistente para importación: contenido de PerformancePoint Server 2007 a SharePoint Server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Si va a mover a SharePoint Server 2013 o para SharePoint Server 2016, es muy probable que necesite crear nuevo contenido del panel (orígenes de datos, informes, cuadros de mandos y páginas de panel).
    
Para empezar a en el plan de actualización de PerformancePoint Services, vea los siguientes recursos:
  
1. [SharePoint Server 2007 fin de soporte: Guía básica](sharepoint-2007-end-of-support.md)
    
2. Cuando sepa qué versión de SharePoint que desea mover a, vea el artículo correspondiente para PerformancePoint Services:
    
  - [Planeación de PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Introducción a PerformancePoint Services en SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Información general sobre PerformancePoint Services en SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
Cuando se actualiza a PerformancePoint Services, podrá disfrutar de varias características nuevas y mejoras. PerformancePoint Services ofrece cuadros de mandos mejoradas, nuevas visualizaciones, como el esquema jerárquico e informe detallado de KPI y más tipos de gráficos, mejores capacidades de filtrado de inteligencia de tiempo y cumplimiento de accesibilidad mejorada. Para obtener más información, vea [What ' s new for PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841343).
